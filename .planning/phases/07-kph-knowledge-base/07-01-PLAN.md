## Phase Goal

**As a** visitor at the klanker-voice booth, **I want to** ask KPH a real question about one of Kurt's systems and get a deep, correct, in-character answer without the conversation feeling slow, **so that** the demo proves KPH actually knows Kurt's world.

## Reconciliation note (read before implementing — carried into execute + verify)

The fat-pack-vs-router tension (CONTEXT D-10/D-11/D-13 vs DESIGN-NOTES Amendment 1) is
resolved as: **router + per-topic deep packs + a stable Kurt-STYLE cached prefix.**
ROADMAP Phase-7 success criterion 2's "retrieval path... from full repo content" is
satisfied by a **bounded classify-then-LOAD-a-pack** (the router picks a pre-baked
per-topic deep pack) — NOT open RAG, document fetch, or tool-calling. This honors D-11's
spirit (no live retrieval) while satisfying criterion 2. Two axes of grounding
(Amendment 2): **WHAT** (facts) = swappable per-topic deep packs after the breakpoint;
**HOW-IT-SOUNDS** (Kurt's style) = the style layer inside the stable cached prefix.

<objective>
Deliver the complete knowledge loop end-to-end on ONE topic (klanker-maker / `km`): a
keyword router classifies a directed question, fires the "OK! Let's dig into it." ack,
swaps the km deep pack into `system[1]` behind a `cache_control` breakpoint on the stable
`system[0]` prefix, and Anthropic prompt caching engages (`cache_read_input_tokens > 0`).
The stable prefix is authored to cross Haiku's 4096-token floor (persona + style layer +
router instructions + all three topic HOOK digests). A km eval scenario proves KPH answers
correctly, and a low-confidence fallback proves the router does not bluff.

Purpose: This is the MVP walking slice — it wires every subsystem (config, pack assembly,
caching, router, pipeline, eval) once, on one topic, so Plan 02 only adds content.
Output: Working router + cached two-block prompt + km content + the two proving tests.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| corpus digest → speakable pack | Corpus digests contain flagged "Landmines" (secrets, PII, infra) that must never reach a pack. |
| anonymous mic user → LLM context | Untrusted spoken input reaches the router + LLM; may attempt prompt injection or force a wrong-topic answer. |
| knowledge pack → public speech | The entire pack is world-readable by design (D-02 "assume full prompt extraction"). |

|-----------|----------|-----------|----------|-------------|-----------------|
| T-07-01 | Information Disclosure | authored km deep pack + hooks | high | mitigate | Author packs from the digest BODY only, excluding every "Landmines / do-not-say" item; Task 2/3 acceptance greps for private-key blocks, AWS-account-id-shaped strings, `.sops`/ARN/internal-hostname markers = zero hits; git-diff review (D-09). |
