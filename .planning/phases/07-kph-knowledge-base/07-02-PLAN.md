<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| spoken utterance → BM25 query | Untrusted text becomes an FTS5 MATCH query — must be parameterized/sanitized, not string-built. |
| corpus files → retrieved chunks → spoken answer | Raw indexed content can surface verbatim in a spoken answer. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-07-05 | Tampering | FTS5 MATCH query built from spoken text | medium | mitigate | Use parameterized queries / FTS5-escape the user text; never string-concatenate into SQL. |
| T-07-06 | Information Disclosure | a retrieved raw chunk surfaces a non-public-but-in-repo detail | medium | mitigate | Corpus is public-by-design (D-02); km index is built from the already-curated on-disk material this plan controls; the curated pack frames chunks; advisory_lint (07-01) + git-diff review (D-09) at refresh time (07-04). |
| T-07-07 | Correctness | retrieval returns off-topic chunks → confident wrong depth | medium | mitigate | Topic-scoped query (never global) + docs-over-source ranking + top_k/token cap. |
| T-07-SC | Tampering | package installs | low | accept | No new packages — sqlite3/FTS5 is stdlib; anthropic already pinned. No legitimacy gate. |
</threat_model>
