|----------|-------------|

| refresh output → committed prompt | Generated packs become world-readable spoken prompt after commit. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-07-10 | Information Disclosure | code-indexing surfaces a public-but-shouldn't-volunteer identifier (e.g. an AWS account id) | high | mitigate | advisory_lint flags account-ids/ARNs/internal-hostnames in the refresh output for the reviewer; D-09 git-diff human review is the gate; packs carry do-not-say sections. |
| T-07-11 | Tampering | manifest points at a non-public source | medium | mitigate | Manifest is curated + reviewed (D-01); the refresh reads manifest-only, never auto-discovers repos; git-diff review before commit. |
| T-07-12 | DoS/Cost | an accidental scheduled/CI refresh burns Anthropic spend | low | mitigate | Refresh is MANUAL only (D-07) — no CI/schedule wiring; cents-per-refresh (D-08). |
| T-07-SC | Tampering | package installs | low | accept | No new packages — reuses anthropic + PyYAML + stdlib sqlite3; transcribe reuses the already-present deepgram SDK. No legitimacy gate. |
