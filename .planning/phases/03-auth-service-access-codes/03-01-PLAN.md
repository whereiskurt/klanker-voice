<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> /api/login | Untrusted email, inviteCode, csrfToken, altcha cross here |
| email link -> /login/confirm -> callback | One-time magic-link token crosses; scanners may prefetch |
| container -> SSM | Secret material (JWT secret, cookie keys, Altcha HMAC, SES creds) loaded at boot |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-01 | Spoofing | Magic-link token consumed by corporate link-scanner (account DoS) | high | mitigate | Interstitial /login/confirm requires explicit click; no auto-submit/onLoad; email link points at confirm page not callback (Task 3, AUTH-01) |
| T-03-02 | Spoofing | Altcha PoW bypass / replay on /api/login | medium | mitigate | Ported verifySolution + in-memory replay LRU (markChallengeUsed); tested in Task 1 (AUTH-05). Note: replay LRU is per-instance — acceptable for PoW at this scale, documented |
| T-03-03 | Info disclosure | Secrets in repo/logs | high | mitigate | Secrets sourced from /kmv/secrets/use1/* via from-aws.tmpl valueFrom; .env/.env.local excluded from the copy; no secret literals committed |
| T-03-04 | Tampering | CSRF on login POST | medium | mitigate | Ported verifyCsrfToken + next-auth csrf cookie retained |
| T-03-05 | Info disclosure | Email enumeration via allowedEmails behavior | low | accept | Public demo; login returns uniform "check your email" message regardless |
| T-03-SC | Tampering | npm install of ported dependencies | high | mitigate | Package Legitimacy Audit (03-RESEARCH.md) satisfied by wholesale-copy-from-production lockfile — all deps installed+running at DEF CON; no [ASSUMED]/[SUS]/[SLOP] packages, so no blocking-human checkpoint required. Build-green check confirms the trim did not break deps |
</threat_model>
