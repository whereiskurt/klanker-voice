<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| pipeline turn text → S3 record | Untrusted user speech + LLM output becomes a durable JSON line; must be JSON-encoded, never string-interpolated |
| access code → code_hash | The raw code (from claim or anon sub) must be reduced to a salted HMAC before any persistence |
| voice task → AWS | Only S3 PutObject; the writer must not reach DynamoDB (LEDG-05) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-15-02-01 | Information Disclosure | raw access code leakage into a record or log | high | mitigate | `code_hash` (HMAC-SHA256, SSM salt) computed once in the constructor; the raw code and any `anon:` sub are never stored in a record dict and never logged (test-asserted: record dict has no raw code / no `anon:` substring; log lines carry counts + keys only) |
| T-15-02-02 | Information Disclosure | transcript text in CloudWatch (accidental second ledger) | medium | mitigate | Flush error path logs record COUNT + key only, never `text`; no INFO-level record logging (acceptance grep) |
| T-15-02-03 | Tampering | quota/transcript co-mingling | medium | mitigate | ledger.py imports no DynamoDB and references no `kmv-voice-usage` (grep-gated → LEDG-05) |
| T-15-02-04 | Denial of Service | ledger I/O stalling a live conversation | medium | mitigate | Flush runs in `asyncio.to_thread`; all exceptions caught and logged (observers.py posture); failed batch re-buffered with a bounded cap |
| T-15-02-SC | Tampering | Python installs | low | accept | No new Python package — boto3 already present via session.py/quota.py; no supply-chain surface added |
</threat_model>
