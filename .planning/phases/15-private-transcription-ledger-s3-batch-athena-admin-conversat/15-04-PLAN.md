## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public internet → ledger bucket | The bucket holds transcripts (personal data); it must be fully private (PAB + SSE, no public policy) |
| voice/auth task roles → ledger bucket | Least privilege — voice writes only, auth reads only, nobody deletes (append-only posture) |
| SOPS → SSM → container | The code_hash salt must never exist in plaintext in the repo or a TOML |
