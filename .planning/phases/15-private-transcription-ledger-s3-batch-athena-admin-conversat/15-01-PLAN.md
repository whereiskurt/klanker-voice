<objective>
Add two namespaced claims — `email` and `code` — to the OIDC access token the auth app issues, and stamp the redeemed access code (`activeCode`) onto AuthProfile on every login. Today the access token carries ONLY `tier_id` + `group`; email lives on the AuthProfile DynamoDB item and the redeemed code lives only on LoginIntent/CodeRedemption items. Without these claims, every magic-link ledger row would have `email=null, code_hash=null` and backfilling is impossible (the token is gone) — RESEARCH Pitfall 4.

Purpose: give the voice service everything it needs to build a complete ledger record from the validated token alone, uniformly across magic-link, bypass, and PSTN paths.
Output: the access token issued by auth.klankermaker.ai carries `https://klankermaker.ai/email` and `https://klankermaker.ai/code`; AuthProfile persists `activeCode` (latest-wins).
</objective>
