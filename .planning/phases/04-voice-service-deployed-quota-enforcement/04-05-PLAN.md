<objective>
Make sessions end gracefully and clean up after themselves. Fill the warning/goodbye callback hooks
04-04 left on the service timer: at -30s inject a high-priority instruction so the concierge speaks the time
warning naturally (D-04), and at 0 push a deterministic goodbye straight to TTS (bypassing the LLM), let it
finish within a ~5s grace, then hard-close and release the slot (D-04/D-05). The same wind-down fires on
mid-session daily/period exhaustion. Then add the three idle-teardown layers (transport disconnect, user-silence
VAD timeout, pipeline stall) atop the wall-clock bound, and a short reconnect grace so a network blip doesn't
kill a live conversation (D-06/D-07).

Purpose: QUOT-03 (spoken warning + graceful goodbye incl. daily exhaustion) and QUOT-05 (layered idle teardown
with reconnect grace and wall-clock outer bound).
Output: the wind-down + teardown logic in session.py/pipeline.py, config knobs, tests, and a listen-and-confirm
checkpoint.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| session lifetime -> metered API burn | a session that never tears down leaks spend and strands a concurrency slot |
| transport reconnect -> session identity | a reconnect must rejoin the same session, not let an attacker hijack another's slot |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-12 | Denial of service | Abandoned sessions never releasing -> stranded slots + runaway spend | high | mitigate | Three independent idle-teardown layers atop the wall-clock cutoff, all routing through an idempotent release() that frees the lease; the heartbeat TTL self-heals even if release() never runs (Task 2) |
| T-04-18 | Spoofing | Reconnect grace lets a different client rejoin an in-flight session | medium | mitigate | Reconnect is keyed to the same session/pc identity and still bounded by the Phase-3 token; the grace only cancels teardown for a matching reconnect, not an arbitrary new offer (Task 2) |
| T-04-19 | Tampering | Goodbye/warning copy injected with untrusted content | low | accept | Copy is operator-authored config, not user input; the deterministic goodbye bypasses the LLM so there is no prompt-injection path through it (Task 1) |
</threat_model>
