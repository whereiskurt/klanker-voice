<objective>
Turn the Phase-1 local pipeline into a deployable, security-gated WebRTC service. Create the production
FastAPI/uvicorn entrypoint (`server.py`) that pipecat's dev runner does not provide for self-hosted Fargate:
it serves POST `/api/offer` (SmallWebRTC signaling) and GET `/health`, validates the Phase-3 JWT access token
offline before creating any session, self-advertises the task's public IP as an ICE candidate with a STUN
srflx backup, and containerizes into a python:3.12-slim image. This is the code half of INFR-03 — the deploy
delta (04-02) and the deployed ICE/RTP proof (04-03) build on it.

Purpose: INFR-03 (WebRTC works deployed) — the transport + auth seam every later session rides on.
Output: server.py, auth.py, webrtc.py, a Dockerfile, and unit tests for token validation and candidate
gathering.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> /api/offer (public internet -> ALB -> task) | untrusted SDP + bearer token cross here; the token is the only thing standing between a public mic and metered APIs |
| task -> AWS control plane (task metadata, EC2 DescribeNetworkInterfaces) | the task reads its own network identity to advertise a candidate |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-01 | Spoofing | Forged / expired / wrong-audience JWT presented at /api/offer | critical | mitigate | validate_access_token does full offline RS256 verification (JWKS signing key, issuer, audience=voice, exp) before any transport is created; Task-1 tests assert forged/expired/wrong-aud/wrong-iss/unknown-key all raise AuthError -> 401 (Task 1, Task 3) |
| T-04-02 | Information disclosure | Token or claim material leaking into logs / error bodies | medium | mitigate | AuthError carries a terse reason only; token bodies are never logged; /api/offer 401 body is a fixed typed string (Task 1, Task 3) |
| T-04-05 | Spoofing | Smoke/service-credential guessed or reused to bypass accounting | high | mitigate | KMV_SMOKE_SERVICE_TOKEN is an SSM-sourced high-entropy secret compared constant-time; bypass sets only bypass_accounting and is honored explicitly by 04-04, never implied; its blast radius (quota bypass) is bounded because it still traverses /api/offer transport (Task 1) |
| T-04-03 | Elevation of privilege | Over-broad task IAM for the EC2/metadata lookup | low | accept | The candidate lookup needs only ec2:DescribeNetworkInterfaces (granted narrowly in 04-02); documented, read-only |
| T-04-SC | Tampering | pip install of PyJWT + boto3 | low | mitigate | Both are on the CLAUDE.md audited/pinned stack (PyJWT 2.13.0 PyPI-verified; boto3 the official AWS SDK); no [ASSUMED]/[SUS]/[SLOP] package, no human legitimacy checkpoint required |
</threat_model>
