<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator -> control item (DynamoDB) | flipping this item pauses/resumes the whole public service |
| kv item keys <-> voice/webapp reads | a key-format mismatch silently hides usage or a stuck kill-switch |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-11 | Tampering | kv writes a control item the voice start-gate cannot read (key mismatch) -> kill-switch silently ineffective | high | mitigate | usage_keys.go reproduces the 04-04 Usage templates with entity/version markers and is asserted equal by table tests; the deployed checkpoint proves an on flip actually rejects a live session (Task 1, Task 2, checkpoint) |
| T-04-20 | Elevation of privilege | kv run with over-broad AWS creds mutating unrelated tables/services | low | accept | Operator tool run by a trusted operator with scoped creds; --usage-table defaults to the voice usage table only; documented |
| T-04-21 | Repudiation | Auto-trip engaged with no record of why/when | low | mitigate | on writes reason + timestamp; kv killswitch status surfaces reason + ceiling so an auto-trip is attributable and requires an explicit operator off to reset (Task 2) |
| T-04-SC | Tampering | go get for usage/killswitch | low | mitigate | Uses only the already-present cobra + aws-sdk-go-v2 dynamodb modules (in kv/go.mod since Phase 3); no new dependency |
</threat_model>
