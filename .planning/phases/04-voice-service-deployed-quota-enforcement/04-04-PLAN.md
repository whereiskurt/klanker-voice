<objective>
Wrap every session in race-safe quota enforcement. Define the DynamoDB usage data model (heartbeat lease,
daily-per-user usage, global rollup, kill-switch control item), then implement the start-gate at /api/offer
(the four typed rejects + sub-floor block + per-task soft cap + atomic heartbeat acquisition) and the
per-session lifecycle (service-timer cutoff, 15s durability/accounting tick, hard-stop at cap, ActiveSessions
metric, and ECS scale-in protection). The tick also updates the global daily rollup and auto-engages the
kill-switch when the site-wide ceiling is crossed. This fills the start_gate seam 04-01 left and provides the
service timer + exhaustion hooks 04-05 builds the spoken wind-down on.

Purpose: QUOT-01 (start blocking), QUOT-02 (conditional-write ticks + hard-stop), QUOT-04 (auto-trip + rollup),
INFR-06 (ActiveSessions metric + scale-in protection + per-task cap).
Output: usage.ts entity, quota.py, session.py, the wired start_gate, quota config knobs, and their tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authenticated user -> quota state (DynamoDB) | a valid token still must not exceed its metered allotment; races are the exploit surface |
| session lifecycle -> site-wide budget | uncontrolled sessions burn the ~$120-165/mo API cap |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-07 | Elevation of privilege | Concurrency-race: two simultaneous offers acquire the same last slot (quota bypass) | high | mitigate | The slot is a conditional-write heartbeat lease (acquire-or-renew only when absent/expired); count-then-acquire is atomic on the write, not a read-check-write; test asserts the second concurrent acquire beyond max_concurrent fails (Task 1, Task 2) |
| T-04-08 | Elevation of privilege | Usage tick lost / spoofed so seconds_used under-counts (free time) | high | mitigate | Ticks are conditional writes with the service-timer wall clock as the authoritative cutoff (D-02) so a dropped tick still hard-stops at cap; hard-stop does not depend on tick durability (Task 3) |
| T-04-09 | Denial of service | Runaway site-wide spend on a public mic | high | mitigate | Global rollup + auto-trip flips the kill-switch at the configured ceiling; per-task soft cap + max_capacity 4 bound concurrent burn (Task 1, Task 3) |
| T-04-10 | Spoofing | Smoke bypass abused to run unmetered real sessions | medium | mitigate | bypass_accounting is accounting-only and gated on the SSM smoke credential (04-01); it never skips transport-layer auth and is covered by idle teardown (04-05) (Task 2, Task 3) |
| T-04-11 | Tampering | Kill-switch control item read stale, letting sessions start during a pause | medium | mitigate | The control item is read on every /api/offer start gate (hot-path read, D-08) and flipped by conditional write, giving near-instant propagation (Task 2) |
</threat_model>
