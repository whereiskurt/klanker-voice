## Phase Goal

**As a** developer, **I want to** hold a slick, measured, interruption-safe conversation with the KlankerMaker concierge on my laptop using only three provider API keys, **so that** the core conversational value is proven before any cloud infrastructure exists.

<objective>
Build the measurement layer: the latency harness (PIPE-05, D-11, D-13) on top of pipecat 1.5.0's built-in UserBotLatencyObserver, and the named eval scenarios that verify barge-in (PIPE-02), memory (PIPE-03), and the greeting (D-04) from deterministic recorded audio through the real input path. Custom scope stays small per RESEARCH: an observer subclass that serializes, a p50/p95 report writer, scenario YAMLs, and an Anthropic judge factory.

Purpose: The harness is the regression instrument for the rest of the project — the A/B verdicts in plan 04, the Phase 5 HUD, and the eventual CI gate all consume its JSON schema. Stage names must be stable from day one.
Output: observers.py, harness package (report + CLI + judge), five scenario YAMLs, report unit tests, observer wiring in both entrypoints.
</objective>
