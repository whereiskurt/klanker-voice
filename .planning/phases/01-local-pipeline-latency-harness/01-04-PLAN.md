## Phase Goal

**As a** developer, **I want to** hold a slick, measured, interruption-safe conversation with the KlankerMaker concierge on my laptop using only three provider API keys, **so that** the core conversational value is proven before any cloud infrastructure exists.

<objective>
Run the endpointing A/B as a three-arm config matrix (RESEARCH Pattern 4; PIPE-04): (a) Nova-3 + Silero VAD stop_secs sweep, (b) Nova-3 + SmartTurn v3 default, (c) Deepgram Flux (eot_threshold). Measure each arm with the plan-03 harness over the same scripted scenario suite, record verdicts with tables and reasoning in docs/TUNING.md (D-12), and land the winning values in pipeline.toml — tuning voice-to-voice toward the ~800ms typical / 1.2s ceiling target (PIPE-01, informational per D-13).

Purpose: This is the measured half of the phase's success criteria — endpointing owns the largest share of the latency budget (PITFALLS #1), and the verdict here is inherited by prod config in Phase 4.
Output: three committed arm configs, three committed harness JSON artifacts, docs/TUNING.md with verdicts, updated pipeline.toml.
</objective>
