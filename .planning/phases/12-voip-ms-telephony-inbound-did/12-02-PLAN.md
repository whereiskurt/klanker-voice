<objective>
Build the D-02 §23 caller-ID mint path in the auth app by mirroring the shipped bypass `/join` machinery: a shared E.164 normalization helper, a `phone` attribute + sparse `byPhone` GSI + `resolvePhoneToCode` resolver on the `AccessCode` entity, and a private, internal-only `GET /tel/<e164>` route that resolves a normalized caller ID → code → tier and mints an OIDC token via the existing `mintAnonToken`. Every piece has a direct analog (`normalizeCode`, `byBypassToken`, `resolveBypassToken`, `/join` route) — this reuses the path so the minted token validates in the voice service unchanged.

Purpose: The telephony controller (12-06) needs a token-minting seam that maps a phone number to the caller's entitled tier while preserving the no-oracle contract and never being internet-exposed (D-02).
Output: normalization helper + entity GSI/resolver + private /tel route, each with unit tests.
</objective>
