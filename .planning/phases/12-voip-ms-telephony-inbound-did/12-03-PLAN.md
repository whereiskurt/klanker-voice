<objective>
Deliver the D-05 operator write path: a `kv code phone <code> --add <e164> | --remove` sub-command that SETs/REMOVEs the `phone` attribute and the sparse `gsi3pk`/`gsi3sk` byPhone key attributes on an access code, plus the electro `AccessCodeGSI3PK`/`AccessCodeGSI3SK` key writers — direct mirrors of the shipped `kv code bypass` + gsi2 key machinery. This is the write side of the §23 caller-ID mapping the auth app (12-02) reads.

Purpose: Operators must be able to map a caller's phone to an existing code so the /tel path resolves it. The Go-side E.164 normalization must produce the byte-identical canonical form the auth-app helper produces, or the byPhone lookup silently misses (Pitfall 3).
Output: gsi3 key writers + `kv code phone` sub-command + Go tests including a normalization parity check.
</objective>
