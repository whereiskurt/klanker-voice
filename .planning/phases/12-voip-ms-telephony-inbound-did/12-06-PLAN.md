<objective>
Wire the §23 caller-ID identity source into the Asterisk controller IN FRONT of the unchanged Phase-11 §24 gate (D-02/D-05): on StasisStart, normalize the caller ID, call the private `/tel` endpoint to mint a token, derive the caller's entitled tier from it, and thread that tier as what the gate grants on unlock — so caller-ID alone never grants a high tier (the gate is still the only unlock), and an unknown/failed caller fails closed with a static goodbye and clean hangup, burning no metered quota. Add the `/tel` endpoint URL + shared auth-token config fields (D-04), and extend `CallIdentity` to carry the resolved tier/caller_id/did.

Purpose: This composes the new identity source with the existing gate seam without modifying the gate, media, or runtime (CONTEXT scope). The controller becomes the point where a phone number becomes a bounded, gate-gated tier grant.
Output: CallIdentity extension, /tel config fields, controller mint wiring + fail-closed, and controller/config tests.
</objective>

|----------|-------------|
| ARI caller-ID field → controller | spoofable caller ID; used for identity only |
| controller → /tel endpoint | Bearer-authenticated internal call over private network |
| minted token → auth.py validation | JWT validated locally before tier is trusted |
