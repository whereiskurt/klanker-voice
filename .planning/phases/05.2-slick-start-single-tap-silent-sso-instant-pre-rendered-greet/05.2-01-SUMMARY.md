# Phase 05.2 Plan 01: Single-Tap Silent SSO Summary

**Guarded, top-level `prompt=none` silent SSO on load — a returning device's tap no longer spends a redirect on sign-in — backed by a token-free localStorage/sessionStorage breadcrumb pair, with a safe `login_required` degrade path.**

## Performance

- **Duration:** ~12 min (first commit to last commit, 2026-07-06T20:35:31-04:00 → 20:38:21-04:00)
- **Started:** 2026-07-07T00:34:00Z (approx, first file read)
- **Completed:** 2026-07-07T00:39:00Z
- **Tasks:** 4/4
- **Files modified:** 12 (7 created, 5 modified, incl. package.json/package-lock.json)

## Accomplishments

- Token-free "returning user" breadcrumb (`returningStore.ts`): `markReturningUser`/`isReturningUser`/`clearReturningUser` (localStorage `kmv_returning`) + `markSilentTried`/`wasSilentTried` (sessionStorage `kmv_silent_tried`, the per-load redirect-loop guard) — every access wrapped in try/catch so disabled storage just skips silent SSO rather than throwing.
- `buildAuthorizeUrl` gained an optional `prompt?: "none"` param — omitted by default (interactive sign-in unchanged), `&prompt=none` only when explicitly requested (silent SSO).
- `useAuth().attemptSilentSso()`: no-op unless (returning user) AND (not authenticated) AND (not already tried this load) — otherwise marks silent-tried, stashes fresh PKCE, and does a top-level navigation to the prompt=none authorize URL. `signOut()` now also clears the returning-user breadcrumb.
- `App.tsx` kicks the guarded `attemptSilentSso()` once on mount (safe to call unconditionally — internally guarded) and sets the breadcrumb on the interactive-return path (`handleAuthenticated`).
- `Callback.tsx` routes `error=login_required` / `interaction_required` onward with NO error UI, clearing the breadcrumb (expected "session expired" degrade, not a failure) — and sets the breadcrumb on the successful code-exchange path too.
