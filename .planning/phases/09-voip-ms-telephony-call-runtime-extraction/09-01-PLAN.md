<objective>
Extract a transport-neutral shared call runtime — `apps/voice/src/klanker_voice/call_runtime.py` — from `server.py`, so both the existing WebRTC `/api/offer` path and future telephony (Phases 10–14) construct, run, and idempotently close ONE live voice session through the same seam. This is a **behavior-preserving refactor** (spec Phase A, §6 / §19-A / §21): the browser voice experience is unchanged.

The extraction moves the current `_run_session` (server.py L149) + `_start_and_run_tracked_session` (server.py L259) bodies into a narrow `create_call_session(...)` / `CallSession` API around an arbitrary pipecat `BaseTransport` (D-01, D-02), and rewires the WebRTC `_connection_callback` (server.py L347) to call it. The WebRTC-specific pieces — `_negotiate_webrtc`, `_connection_callback`'s signaling, `_wire_connection_teardown`, `_extract_bearer_token`, the `/api/offer` routes, the SPA mount — STAY in server.py (D-03).

Purpose: the whole telephony milestone hinges on reusing the pipeline/lifecycle/quota wiring with a different transport at the edge; extracting the reusable seam FIRST (zero behavior change, full test coverage) de-risks every later phase and prevents duplicated lifecycle/pipeline code.

Output: `call_runtime.py` (new module), a rewired `server.py`, `tests/test_call_runtime.py` (new focused tests), and a short architecture/coupling note (D-08). Stop after Phase A (spec §21.6/§21.10) — no SIP/RTP/codec/infra.
</objective>
