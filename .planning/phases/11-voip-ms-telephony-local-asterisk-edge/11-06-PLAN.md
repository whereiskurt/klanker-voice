<objective>
Implement the silent §24 answer-gate (D-05): one persistent pipeline with an inline `GateProcessor` (R5) that keeps the agent dark until the caller proves access via DTMF PIN (controller layer) or a spoken order-independent 4-word passphrase (STT matcher, outside the LLM), then grants tier and runs the normal greet→LLM→TTS path; fail-closed goodbye+hangup on timeout.

Purpose: This is the primary DoS/abuse control for a DEF-CON-hostile public phone line (§25.D). The redaction boundary — pre-unlock transcript never reaches the LLM/ledger/logs — is enforced structurally by never forwarding those frames (R5, Landmine 6). Secrets live only in env; they never enter LLM context or logs (D-05e).
Output: `telephony/gate.py` (GateProcessor + passphrase matcher + DTMF accumulator + fail-closed timer); additive `gate_processor` threading through `build_pipeline`/`create_call_session`; controller wiring of unlock→quota→greet and fail-closed; gate unit tests.
</objective>
