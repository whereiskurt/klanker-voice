<objective>
Implement the socket-backed `RtpMediaSession` (D-03) that drops into the Phase 10 seam without touching the codec or transport: a passive UDP listener that binds first, learns Asterisk's send-from address via symmetric-RTP first-packet source learning, and exchanges raw RTP datagrams behind the exact `read_packet`/`write_packet`/`close` Protocol.

Purpose: This is the telephony analog of `webrtc.py` isolation — a transport-specific media module, not a branch in shared code. External Media only supports `connection_type=client`, so Asterisk always initiates; Klanker must be listening before the externalMedia channel is created (R2).
Output: `SocketRtpMediaSession` + `_AsteriskRtpProtocol` in `telephony/rtp_socket.py`; unit tests over a real localhost UDP loopback.
</objective>
