<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public PSTN/SIP → Asterisk dialplan | Hostile inbound INVITEs (scanners, DEF CON traffic) hit the SIP transport |
| Asterisk ARI HTTP port → network | ARI grants full call control; exposure = full takeover |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-11-02-01 | Elevation of privilege | extensions.conf dialplan | high | mitigate | Single inbound-only context → Stasis; no Dial()/outbound context anywhere (Task 1 + Task 3 lint) — a compromised endpoint cannot place outbound/toll calls (§25.A) |
| T-11-02-02 | Spoofing / Information disclosure | ari.conf / http.conf | high | mitigate | ARI bound to private/loopback + authenticated user; empty allowed_origins; version-pinned image; cloud SG hardening deferred to Phase 14 (§18/§25.C) |
| T-11-02-03 | Denial of service | SIP transport (public entry) | medium | accept | Local dev harness is loopback-only; the gate (Plan 06) + concurrency=1 are the runtime DoS controls; public-edge rate limiting/fail2ban is Phase 14 |
| T-11-02-SC | Tampering | andrius/asterisk docker image | medium | mitigate | Pin the image to a specific version tag (not `latest`); document the tag + module smoke check in README |
</threat_model>
