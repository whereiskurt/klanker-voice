## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /api/offer | Bearer token + SDP offer to metered paid APIs; server validates + quota-gates |
| RTVI stream → UI | server-authored transcript/level/state messages drive the orb + captions |
| getUserMedia | mic access is a sensitive browser capability, gesture-gated |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-04-E | Elevation of Privilege | starting media before authorization | high | mitigate | The client starts NO conversation/media UI until /api/offer returns a successful answer; 401 (bad token) and typed quota rejects (403/429) are exposed as non-connected outcomes — the server start_gate is authoritative and the client never bypasses it. |
| T-05-04-I | Information Disclosure | Bearer token transport | high | mitigate | Token sent only same-origin over HTTPS to /api/offer (Authorization header / request_data.access_token); never logged, never placed in a URL/query. |
| T-05-04-T | Tampering | RTVI-driven UI | low | mitigate | RTVI messages only drive display (orb/captions); they never carry privileges or execute code. Caption text is rendered as text (React escaping), not HTML. |
| T-05-04-D | Denial of Service | orb amplitude loop | low | accept | GPU render loop bounded (05-02); amplitude smoothing is O(1) per frame. |
| T-05-04-SC | Tampering | npm installs | low | accept | No new packages beyond 05-02's vetted @pipecat-ai/* transport set. |
</threat_model>
