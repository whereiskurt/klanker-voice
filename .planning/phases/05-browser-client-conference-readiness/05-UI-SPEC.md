# Phase 5 — UI Design Contract

> Visual and interaction contract for the public voice client at **voice.klankermaker.ai**.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **This is the project's design centerpiece.** The stated acceptance feel is *"whoa in the
> first ten seconds"* — visual + motion quality is a first-class requirement, not decoration.
> Every token, state, and string below is prescriptive so the planner/executor can build the
> immersive orb-centric stage without design ambiguity.

**Source legend:** `[CTX D-nn]` = locked 05-CONTEXT decision · `[REQ]` = REQUIREMENTS/CLNT ·
`[SPEC]` = design spec §Frontend/§Session/§Failure · `[BRAND]` = auth-webapp handoff reference ·
`[REC]` = researcher recommendation within a discretion area (CTX "Claude's Discretion").

## Orb Visual Spec  (CLNT-04 · [CTX D-05/06/07])

The orb is the hero. It is **audio-reactive AND state-colored** — it deforms to live amplitude *and*
carries a distinct color/motion signature per state.

**Render technique — recommendation [REC]:** a single **full-screen WebGL2 fragment-shader** canvas
(one fullscreen quad; the orb is drawn as a signed-distance/metaball field with simplex-noise domain
warping). Rationale: GPU-driven 60fps, near-zero main-thread CPU (leaves the CPU for WebRTC audio
decode + the latency budget), tiny bundle vs a 3D engine, and organic deformation comes free from noise
uniforms. **Accepted alternative:** Three.js icosphere with vertex-displacement if the team prefers
geometry — but default to raw WebGL2 for footprint. **Mandatory fallback:** no-WebGL / `prefers-reduced-motion`
→ 2D `<canvas>` (or CSS) radial-glow that scales gently with amplitude, no noise churn.

**Amplitude → deformation mapping:**
- Drive a `uAmplitude` uniform `[0..1]` from RTVI audio levels — **mic input RMS while listening**,
  **bot TTS output RMS while speaking**. [CTX D-06 — RTVI exposes levels + bot-speaking]
- Smoothing: exponential moving average, **fast attack ~60ms / slow release ~180ms** (feels alive, not jittery).
- Amplitude modulates: orb radius pulse (±8–14%), noise displacement gain, and bloom intensity.

**Per-state motion signature:**

| State | Color | Motion character |
|-------|-------|------------------|
| Idle / attract | aqua `#2DE2C8` | Slow ambient breathe (4–8s loop), low-freq surface drift. **Alive before a word is spoken.** [CTX D-07] |
| Listening | blue `#4EA8FF` | Sharp, responsive spikes tracking mic RMS; surface tightens/ripples to voice |
| Thinking | violet `#8B7CF6` | No external amplitude; internal churn/rotation — a contained "processing" swirl |
| Speaking | aqua-green `#34F5D0` | Confident bloom + smooth deformation tracking TTS RMS; brightest halo |

State color morph: **600ms** ease (see motion tokens); never a hard cut.

## Stage Layout & Composition

Full-viewport (`100dvh`), near-full-bleed, single stage. Orb centered as hero; everything else is a
**restrained overlay** that never competes. [CTX D-05/08/10]

```
┌───────────────────────────────────────────────┐
│  [wordmark]                        ⏱ 01:47  ← countdown pill (top-right)
│                                                │
│                                                │
│                    (  ORB  )   ← hero, centered│
│                                                │
│                                                │
│        ┌───────────────────────────────┐       │
│  KPH ▸ │  agent last utterance (final) │  ← caption band (bottom third)
│  You ▸ │  your last utterance (interim)│       │
│        └───────────────────────────────┘       │
│  ⌁ HUD                        [ Tap to talk ]   │  ← HUD toggle (BL) · mic CTA (lower third)
└───────────────────────────────────────────────┘
     ^ inside safe-area insets
```

| Element | Desktop | Mobile portrait (iPhone) |
|---------|---------|--------------------------|
| Orb | Centered, ~45vmin, slightly above vertical center | Upper-middle, ~55vmin — leaves lower third for captions + CTA (thumb reach) |
| Captions | Centered band, `max-width 720px`, above CTA | Full-width minus 16px, above CTA, ≤2 lines/side |
| Countdown pill | Top-right, `xl` margin | Top-center (avoids notch), safe-area offset |
| HUD toggle | Bottom-left affordance + `h` key | Bottom-left tap; long-press safe zone |
| Mic CTA | Below orb, centered | **Lower third, one-handed reach**, ≥96px hit area |
| HUD panel (open) | Slides from bottom-left, translucent secondary surface | Bottom sheet, translucent |

Captions are **subtitle-style, last-exchange-only** — current/last utterance per side, interim gray
firming to final `#E6E8EF`, fading as conversation moves on. **Not** a rolling transcript log. [CTX D-08]

## Screen & State Flow

Full flow from cold load to reconnect. Each state names its trigger and its dominant UI.

1. **Landing / attract** [CTX D-07] — orb already idling (ambient motion) + one prominent **"Tap to talk"**
   CTA + small wordmark. The "whoa" lands here, pre-conversation. Tapping = the single user gesture that
   unlocks mic + audio playback (iOS) and begins the flow.
2. **Sign-in gate** (CLNT-08) — if no in-memory token, the tap routes to **OIDC authorization-code+PKCE**
   full-page redirect to auth.klankermaker.ai. Show a brief "Signing you in…" redirect state. Return via
   callback route → token held in memory. [CTX D-04]
3. **No-access gate** [CTX D-13] — authenticated but `no-access` tier: exclusive/invite-only panel
   (copy below). Aspirational, not a dead-end.
4. **Mic permission** (CLNT-01) — gesture-gated `getUserMedia`. Distinct inline states: *requesting* /
   *denied* / *no device* / *unsupported browser*. [CTX D-12]
5. **Connecting** (CLNT-02) — "Connecting…"; on ICE/connect failure, **bounded auto-retry + backoff**
   with visible "Reconnecting… (attempt n/N)". [CTX D-11]
6. **Live conversation** — orb reactive; captions both sides; countdown pill; HUD toggle available.
7. **Wind-down** — near `session_max − 30s`, countdown escalates (amber → red `<10s`), **synced to the
   agent's spoken −30s warning**. [CTX D-10, QUOT-03]
8. **Session end** [CTX D-14] — clean end (timer/quota/goodbye): brief summary + one **"Reconnect"**
   button that **re-runs the quota start-gate** before reconnecting.
9. **Honest failure wall** [CTX D-11] — retries exhausted (likely UDP-blocked, no TURN v1): STOP, show
   honest message + manual retry. **No infinite spinner.**
10. **Gate rejections (typed start-gate)** [CTX D-14] — each `quota.GateResult` reject maps to specific
    copy: *daily-exhausted*, *over-concurrent*, *killswitch*. Exclusive-adjacent tone (D-13).

**Destructive / high-consequence actions & confirmation approach:**

| Action | Confirmation approach |
|--------|----------------------|
| **Sign out** | Single tap, no modal (low consequence — token is in-memory only; re-auth is one redirect). [CTX D-04] |

| **Reconnect** | Not destructive; but it **re-runs the quota start-gate** — if rejected, route to the matching gate copy above rather than a raw error. [CTX D-14] |

> No modal confirmation dialogs in v1. The immersive stage avoids interrupting overlays; consequences here
> are low (in-memory token, server-managed session lifecycle).

## Motion Tokens

| Token | Duration | Easing | Applied to |
|-------|----------|--------|------------|
| `motion-instant` | 120ms | `cubic-bezier(0.4, 0, 0.2, 1)` | Caption interim→final firm, chip color |
| `motion-fast` | 200ms | `cubic-bezier(0.4, 0, 0.2, 1)` | Overlay/HUD fade + slide, button hover |
| `motion-base` | 320ms | `cubic-bezier(0.16, 1, 0.3, 1)` (expo-out) | Panel/gate enter, caption fade-out |
| `motion-orb-morph` | 600ms | `cubic-bezier(0.16, 1, 0.3, 1)` | Orb state **color** transition |
| `motion-ambient` | 4000–8000ms loop | sine / eased loop | Idle-attract breathe + surface drift |
| `motion-escalate` | 800ms loop | ease-in-out | Countdown `<10s` red pulse + slight scale |
| Orb amplitude smoothing | attack ~60ms / release ~180ms | EMA | `uAmplitude` uniform from RTVI RMS |

`prefers-reduced-motion: reduce` → disable orb noise churn, ambient drift, and escalation pulse; keep
gentle opacity/scale only; caption + countdown state conveyed by text/color, not motion. [CTX a11y]

## Interaction & Data Binding  (RTVI → UI)

Planner/executor contract for wiring pipecat client-js RTVI events to visuals.

| UI surface | RTVI / client-js source | Notes |
|-----------|-------------------------|-------|
| Orb amplitude (listening) | local mic audio level / RMS | fast-attack EMA → `uAmplitude` |
| Orb amplitude (speaking) | bot audio output level / `bot-speaking` | drives `uAmplitude` + state=speaking |
| Orb state machine | transport connection + STT partial + `bot-speaking` + inflight LLM/TTS | listening ↔ thinking ↔ speaking ↔ idle |
| Captions (user) | user transcript frames (interim → final) | interim gray, firm to final |
| Captions (agent) | bot transcript frames | agent speaker chip = accent |
| Countdown | session start time + tier `session_max_seconds` (from token/`/api/offer`) | client-side countdown; escalate near 0 |
| **Latency HUD** | **RTVI metrics frames — per-stage STT / LLM-TTFT / TTS-first-audio / v2v p50** | ⚠ **`observers.py` computes these server-side but they are NOT yet surfaced per-turn to the client. Emitting them as RTVI metrics frames is a required server-side integration task** for CLNT-06/D-09 — flag prominently in PLAN. |
| Gate rejections | typed `quota.GateResult` from `/api/offer` | map each type → gate copy above [CTX D-14] |
| Connection state | small-webrtc-transport ICE/connection events | drives connecting/reconnecting/wall [CTX D-11] |

**Auth wiring:** authorization-code+PKCE; access token **in memory only** (re-auth on refresh, XSS-safer);
sent as `Bearer` to `/api/offer`; SPA callback route registered against the `voice` OIDC client. [CTX D-04]

## Latency HUD Spec  (CLNT-06 · [CTX D-09])

Off by default (pristine for audiences); revealed by toggle (tap the "Latency" affordance or press `H`).
Translucent secondary-surface panel, **monospace 13px**, one row per stage, updating live per turn:

```
STT            142 ms
LLM  TTFT      318 ms
TTS  1st-audio  76 ms
voice→voice   p50  1.40 s
```

Doubles as the operator's tuning instrument (Phase 6 baseline is ~1402ms p50). Values right-aligned,
mono, `text-secondary` labels / `text-primary` numbers. Escalate nothing — this panel is informational.

## Accessibility Baseline  ([CTX a11y, mobile realities])

- **Captions are the primary a11y affordance for the audio** — both sides always rendered; do not gate
  captions behind the HUD.
- **Contrast:** final caption text ≥ 7:1 on scrim; CTA/label ≥ 4.5:1; never rely on color alone (state,
  countdown, and errors all carry text + `aria-live`).
- **Reduced motion:** `prefers-reduced-motion` swaps the shader orb for the calm fallback and disables
  ambient/escalation motion.
- **Focus & keyboard:** visible focus rings on CTA, HUD toggle, retry/reconnect; `H` toggles HUD; `Esc`
  dismisses transient gate copy where applicable.
- **Touch targets:** mic CTA ≥ 96px hit area; all buttons ≥ 44px; CTA in one-handed lower-third on mobile.
- **Live regions:** connection status and countdown announced via `aria-live="polite"`; errors `assertive`.
