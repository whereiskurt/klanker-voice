Stand up the bespoke Vite + TypeScript + React SPA at `apps/voice/client/`, encode the UI-SPEC design
system as CSS tokens, and build the **hero orb** — the WebGL2 shader plasma + orbiting particle-ring
(sketch 001 winner A) with its mandatory 2D fallback — running the D-07 attract landing so the "whoa"
lands before a word is spoken. Add the D-03 multi-stage Docker build so `dist/` ships inside the deployed
image (served by the StaticFiles mount from 05-01).

Purpose: This is the first user-facing slice — a real, demoable capability (the alive attract orb on the
served stage), and the foundation every later client plan extends. The orb is fully built here (in idle/
attract mode only, since no live audio exists yet); 05-04 drives it with real RTVI amplitude + state.
Output: A served, buildable SPA whose landing screen is the immersive attract stage with the reactive orb.

Context budget note: the orb shader + particle ring already exist as a working reference in
`.planning/sketches/001-immersive-orb-stage/index.html` — this task PORTS that code into a React
component, it does not author a shader from scratch.
</objective>
