<objective>
Wire the already-built voice pipeline to emit everything the browser client needs over the RTVI
data channel, and serve the built SPA from the same FastAPI app. This is the **server half** of
three client requirements: captions (CLNT-03), the state-aware orb (CLNT-04), and the latency HUD
(CLNT-06/D-09). `observers.py` already computes per-stage timings for the local harness but they are
**not surfaced per-turn to the client** — this plan closes that gap by pushing a composed
`RTVIServerMessageFrame` per turn, and adds the standard `RTVIProcessor`/`RTVIObserver` that give
client-js transcripts, bot/user speaking, and audio levels for free.

Purpose: The bespoke React client (05-02…05-07) can only render captions/orb/HUD if the server speaks
the RTVI protocol. No pipeline currently includes an `RTVIProcessor`, so today client-js would connect
and receive audio but no data events. This plan is a Wave-1 prerequisite that unblocks the client chain.
Output: RTVI-enabled pipeline + composed latency emission + StaticFiles SPA mount, all unit-tested.
</objective>

## Phase Goal

**As a** member of the public at a conference, **I want to** sign in at voice.klankermaker.ai, tap the
mic, and hold a slick low-latency conversation with the KlankerMaker concierge — with a live orb, captions,
a countdown, and honest failure handling — **so that** I experience the "whoa" of the demo and can cleanly
reconnect for another turn.
