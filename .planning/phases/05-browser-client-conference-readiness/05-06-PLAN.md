<objective>
Make the client honest and resilient: bounded auto-retry + backoff on connection failure that ends at a
clear **UDP-blocked wall** (no infinite spinner, no TURN in v1 — D-11/CLNT-02); map each **typed start-gate
rejection** (daily-exhausted / over-concurrent / killswitch) to its specific in-client gate copy (D-14); and
deliver a **clean session end + one-click Reconnect that re-runs the quota start-gate** before reconnecting
(CLNT-07/D-14).

Purpose: This is success criteria 2 (honest ICE/UDP messaging + retry) and 5 (clean end + quota-rechecked
reconnect). It consumes the typed `quota.GateResult` error_types the server already returns and the
transport lifecycle events from 05-04. Output: the failure/gate/end/reconnect UX that makes the demo robust
on hostile conference networks.
</objective>

## Phase Goal

**As a** member of the public at a conference, **I want to** sign in at voice.klankermaker.ai, tap the
mic, and hold a slick low-latency conversation with the KlankerMaker concierge — with a live orb, captions,
a countdown, and honest failure handling — **so that** I experience the "whoa" of the demo and can cleanly
reconnect for another turn.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| /api/offer typed rejects → client | server-authoritative quota/killswitch verdicts the client must honor |
| Reconnect → /api/offer | re-entry into the metered pipeline must re-run the start_gate |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-06-E | Elevation of Privilege | Reconnect bypassing quota | high | mitigate | Reconnect issues a FRESH /api/offer that re-executes the server start_gate — it never silently reopens a transport; a quota reject routes to a gate, so a user cannot reconnect past their cap or the killswitch. |
| T-05-06-E | Elevation of Privilege | retrying a rejection as a network blip | high | mitigate | Code strictly separates transport failure (retry/wall) from authorization/quota rejection (gate); 401/403/site-paused are never auto-retried, so retry cannot brute-force past the gate. |
| T-05-06-I | Information Disclosure | gate/error copy | low | mitigate | Gate cards render only the friendly message from {error,message}; the raw exception detail path is not shown to users. |
| T-05-06-SC | Tampering | npm installs | low | accept | No new packages. |
</threat_model>
