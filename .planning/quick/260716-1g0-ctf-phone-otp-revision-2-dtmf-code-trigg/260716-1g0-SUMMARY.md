# Quick Task 260716-1g0: CTF Phone-OTP Revision 2 -- DTMF-Code Trigger Summary

**Replaced the dead Revision-1 DID-keyed pre-gate CTF-OTP announcement with a DTMF access-code trigger checked inside the existing §24 answer-gate, adding `GateProcessor.cancel_for_takeover` so the redaction boundary stays closed during takeover.**

## Performance

- **Duration:** ~45 min
- **Tasks:** 2/2 completed
- **Files modified:** 7 (2 source config, 2 source logic, 3 test files) across 2 commits

## Accomplishments

- `AnnouncementEntry.code_env_var` is now REQUIRED at config load (holds only the env-var NAME carrying the DTMF trigger code, e.g. `CTF_ANNOUNCEMENT_CODE`; the trigger value itself lives only in SSM, never TOML). `did` became optional/informational (Revision 1's root-cause bug: VoIP.ms routes every DID to one sub-account, so the dialed number was never visible at `on_stasis_start`).
- `apps/voice/configs/telephony.toml`'s single `[[telephony.announcement]]` entry was re-keyed to `code_env_var = "CTF_ANNOUNCEMENT_CODE"`; the old `did = "7254043234"` matcher line is now a commented informational note.
- `AsteriskCallController.on_channel_dtmf_received` now checks the real `TELEPHONY_ACCESS_PIN` FIRST (byte-unchanged priority, `return`s immediately on a match), then — only on a PIN miss — suffix-matches a separate bounded raw digit buffer (`ActiveCall.dtmf_raw`, `DTMF_RAW_MAX_DIGITS=32`) against every armed announcement code.
- New `AsteriskCallController._gate_announcement(active_call, entry)` mirrors `_gate_fail_closed`'s exact shape: `GateProcessor.cancel_for_takeover("announcement")` → `_fetch_ctf_otp` → on failure `_close_active_call` with no spoken line; on success `speak_goodbye(...)` with the digit-spaced twice-substituted line → `ANNOUNCEMENT_PLAYBACK_GRACE_SECONDS` sleep → `_close_active_call`. Never calls `quota.start_gate` or `greet_now`.
- New `GateProcessor.cancel_for_takeover(reason)` (gate.py): resolves the gate (`_resolved = True`, cancels the fail-closed timer) WITHOUT setting `_unlocked` — the §24 redaction boundary (D-05e) stays closed the entire time the announcement plays, and the fail-closed timer can never race the announcement's own goodbye. Idempotent; logs only `reason` + `call_id`.
- Removed the dead Revision-1 code path entirely: `_announcements_by_did`, the pre-gate `on_stasis_start` dispatch block, and the `_run_announcement` method — along with the now-unused `Pipeline`/`WorkerRunner`/`build_tts`/`build_worker`/`TTSSpeakFrame`/`contextlib` imports in controller.py.
- Stripped the last literal `990011` occurrences from `telephony.toml` and `config.py` docstrings (the redaction spot-check requires zero literal trigger-code occurrences in TOML/src — only the env-var NAME is ever written down).

## Decisions Made

- **`_gate_announcement` reuses the persistent pipeline's existing worker via `speak_goodbye`, not a standalone TTS-only pipeline.** Revision 1's `_run_announcement` built a fresh `Pipeline([tts, transport.output()])` + `WorkerRunner` because it ran BEFORE the gate/pipeline existed. Revision 2 fires from inside an already-running gated pipeline (the call is mid-gate-window), so mirroring `_gate_fail_closed`'s existing `speak_goodbye(active_call.call_session.worker, line)` pattern is both simpler and consistent with the design doc's own "mirrors `_gate_fail_closed`" language. This let `Pipeline`/`WorkerRunner`/`build_tts`/`build_worker`/`TTSSpeakFrame`/`contextlib` all be removed as dead imports — a net simplification, not scope creep.
- **`cancel_for_takeover` is synchronous**, matching the plan's `-> None` signature (no `Awaitable`) — it does no I/O, only flag flips and `Task.cancel()`, unlike `unlock()`/`_fire_fail_closed()` which `await` a caller-supplied callback.
- **Test spec deviation:** the plan's task-2 behavior bullet (d) mentioned "monkeypatch `build_tts` to raise" as part of the OTP-fetch-failure test. Since the actual `_gate_announcement` implementation never references `build_tts` (see decision above — it reuses the existing worker), that monkeypatch target no longer exists in controller.py's namespace. The equivalent guarantee ("no TTS/spoken line on fetch failure") is instead asserted via a `speak_goodbye` spy that raises `AssertionError` if called — functionally identical coverage, adapted to the actual (simpler, design-doc-faithful) implementation.
