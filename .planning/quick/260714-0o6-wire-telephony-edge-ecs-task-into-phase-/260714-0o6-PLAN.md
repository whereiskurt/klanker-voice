# Quick Task 260714-0o6: Wire telephony-edge into the transcription ledger

## Problem

PSTN phone-call transcripts are being **silently dropped**. Root cause confirmed
live before planning (no re-investigation needed):

- The Phase-15 ledger writer (`apps/voice/src/klanker_voice/ledger.py`) runs
  **in-container on the telephony-edge task** — PSTN calls hit the same
  `create_call_session()` ledger tap as WebRTC.
- `_bucket()` reads env var `KMV_LEDGER_BUCKET` (default `""`).
- The terragrunt ledger-bucket injection block covered only `voice` and
  `auth`, **never `telephony-edge`**. The live telephony-edge task def (:12)
  had zero LEDGER env vars.
- Result: `put_object(Bucket="")` → boto3 `Parameter validation failed` on
  every flush. Observed in `/ecs/telephony-edge` at 03:31 / 04:13 / 04:17 UTC
  on 2026-07-14. WebRTC calls are unaffected (they run on `voice`, which is wired).

## Fix (mirror voice's ledger wiring onto telephony-edge)

Three interdependent parts — all necessary, none sufficient alone:

1. **Bucket env** — `infra/terraform/live/site/region/us-east-1/ecs-task/terragrunt.hcl`:
   add a `task.name == "telephony-edge"` branch to the container-env injection
   ternary, injecting `KMV_LEDGER_BUCKET = dependency.ledger.outputs.bucket_name`
   (same var/value as voice).
2. **Salt secret** — `infra/terraform/live/site/services/telephony-edge/service.hcl`:
   add the `KMV_LEDGER_SALT` secret (`valueFrom` the shared SSM SecureString
   `/kmv/secrets/use1/ledger/code_hash_salt`), so a PSTN caller who unlocks with
   an access code gets a correctly-salted `code_hash` matching WebRTC rows.
3. **PutObject IAM** — same `service.hcl`: add a `LedgerPutOnly` statement
   (`s3:PutObject` on `arn:aws:s3:::kmv-ledger-*/ledger/*`) to
   `task_role_iam_statements`. Without it the PUT would `AccessDenied` even
   once the bucket env is set. PutObject-only, matching voice's append-only posture.

Plus a parity `NOTE` comment in the telephony-edge env block (mirrors voice)
so this regression can't silently recur.

## Verify
