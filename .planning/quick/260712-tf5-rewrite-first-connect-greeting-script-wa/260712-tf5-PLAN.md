# Quick Task 260712-tf5: Rewrite first-connect greeting (warmer KPH intro)

## Goal

The pre-rendered first-connect greeting sounded flat. Replace it with a warmer,
more characterful take — iterated live with Kurt via an ElevenLabs audition loop
(render candidates at the exact live `pipeline.toml` settings → Kurt listens →
tweak → repeat) until approved.

## Approach (as executed — planning happened interactively in-session)

1. **Audition loop** (scratchpad, no repo changes): ~20 candidate renders across
   script wording, `<break>`-tag pause lengths, and stability/style knobs, all at
   live prod voice settings (Burt Fundeck, eleven_flash_v2_5, speed 1.02 /
   stability 0.45 / style 0.3). Delivery variance handled by re-rolling identical
   text and cherry-picking, then ffmpeg-splicing the best segments at break-tag
   silences (front of K-take-3 + "Ah — his Klanker!" tail + fresh punchy ending).
2. **Ship** the approved clip (`M-final-3-plain`):
   - `client/public/greetings/greeting-1.mp3` ← the spliced clip (7.2s)
   - `greetings.source.json` → single canonical script (with break tags)
   - `greetings.manifest.json` → matching single clip + do-not-re-render note
   - delete `greeting-2.mp3` / `greeting-3.mp3` (rotation collapsed to one
     approved take; client handles 1-clip manifests)
3. **Verify**: drift-guard + full pytest suite.

## Final approved script

> Hey! `<break 0.6s>` You've reached K P H. `<break 0.7s>` Kurt's virtual
> assistant. `<break 0.8s>` Ah — his Klanker! `<break 0.8s>` How can I help?

## Key decisions

- **Cherry-picked clip over reproducible render**: Flash delivery is
  nondeterministic; the greeting ships as a static asset, so the approved take is
  committed directly. Manifest `note` field warns against casually re-running
  `make greetings` (it would replace the approved delivery).
- **Single greeting, no rotation**: only one take earned approval; variants can
  be re-added later by appending to source.json + re-auditioning.
- **Spelled-out "K P H"** for sharp letter delivery; break tags (not ellipses)
  for pauses — trailing ellipses caused the end-of-clip volume fade.
