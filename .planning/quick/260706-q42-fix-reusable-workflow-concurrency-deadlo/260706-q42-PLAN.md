# Quick Task 260706-q42: Fix reusable-workflow concurrency deadlock in deploy.yml

## Problem (root-caused live)

`build-voice.yml`, `build-auth.yml`, and `deploy.yml` all declare:

```yaml
concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: false
```

`deploy.yml` is called as a **reusable workflow** (`uses: ./.github/workflows/deploy.yml`)
from the `deploy` job of `build-voice.yml` / `build-auth.yml`. Inside a called workflow,
`${{ github.workflow }}` resolves to the **caller's** workflow name (e.g. "Build: Voice"),
so the called `deploy` job requests the *same* concurrency group its parent run already
holds → GitHub aborts it as a **deadlock**. The deploy job never materializes and the run
goes red even though the build job succeeded (empirically confirmed: run 28827809814 had
1 job = build success, run conclusion = failure, no deploy check-run).

## The fix (single task)

Give `deploy.yml` a **distinct literal concurrency group** that cannot collide with any
caller's `github.workflow`-derived group. Change ONLY `deploy.yml`'s `concurrency.group`:

```yaml
concurrency:
  group: deploy-ecs-${{ github.ref }}
  cancel-in-progress: false
```

Rationale for `deploy-ecs-${{ github.ref }}`:
- Distinct from "Build: Voice-*" / "Build: Auth-*" → no parent/child deadlock.
- Still per-ref, and shared across voice+auth deploys with `cancel-in-progress: false`,
  which correctly **serializes** concurrent deploys onto the shared `ecs-task`/`ecs-service`
  terragrunt state (desirable — prevents two `terragrunt apply`s racing the same state).
- Works identically for the `workflow_dispatch` path (standalone deploys) — no behavior change there.

Do NOT touch `build-voice.yml` / `build-auth.yml` concurrency (their groups are correct for
their own runs). Do NOT change any other part of `deploy.yml`.

### Task 01
- **files:** `.github/workflows/deploy.yml`
- **action:** Replace the `concurrency.group` value `${{ github.workflow }}-${{ github.ref }}`
  with `deploy-ecs-${{ github.ref }}`. Leave `cancel-in-progress: false` unchanged.
- **verify:** `grep -A2 '^concurrency:' .github/workflows/deploy.yml` shows
  `group: deploy-ecs-${{ github.ref }}`; confirm `build-voice.yml` and `build-auth.yml`
  concurrency blocks are unchanged (`git diff` touches only deploy.yml). Optionally validate
  YAML parses (`python -c "import yaml,sys; yaml.safe_load(open('.github/workflows/deploy.yml'))"`).
- **done:** deploy.yml has the distinct group; only deploy.yml changed.
