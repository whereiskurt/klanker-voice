<objective>
Part A of the per-DID gate policy: make the §24 answer-gate per-DID. The two Las
Vegas DIDs (`7254043234` / `7254043283`) become OTP-only — the concierge
passphrase and concierge DTMF PIN do NOTHING, only the global `333266` OTP
announcement works, and a call with no code falls through to the existing
fail-closed hangup. Every other DID (NYC / Toronto-Belleville / Caldwell, all of
which resolve to `dialed_did=<none>`) stays exactly today's concierge + OTP
behavior.

The design is fully specified in
`docs/superpowers/specs/2026-07-17-per-did-gate-policy-and-cid-tooling.md`
("Implementation (real seams, verified 2026-07-17)", 6 numbered steps). The seams
are verified — do NOT re-derive the design.

Purpose: only the 2 Vegas DIDs carry a CID prefix, so only they resolve a
`dialed_did` at the edge. Inverting on that already-identifiable set means zero
new VoIP.ms work for the concierge DIDs.
Output: a per-DID `otp_only_dids` config, an `otp_only` flag threaded through the
gate/controller, concierge-unlock suppression that leaves the 333266 takeover and
fail-closed paths untouched, and tests proving both the OTP-only and the
byte-identical concierge behavior.

This is Part A ONLY. Parts B (kv CID-prefix tooling) and C (kv studio surface)
are explicit follow-ons — do NOT plan or implement them here.
</objective>

| VoIP.ms CID name prefix → dialed_did | Caller-influenced `${CALLERID(name)}` is the basis for the per-DID policy decision. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-O2Q-01 | Elevation of Privilege | GateProcessor concierge suppression | high | mitigate | Suppression is enforced at BOTH layers (gate.py `unlock` no-op for passphrase/dtmf + process_frame skips accumulation; controller skips the PIN branch) so no single missed path re-opens concierge on a Vegas DID. Tests 1+2 assert both factors inert. |
| T-O2Q-02 | Spoofing | dialed_did via caller-supplied CALLERID(name) | low | accept | A spoofed prefix can only move a call INTO the more-restrictive OTP-only bucket or leave it in the default concierge bucket; it cannot grant access (333266 + fail-closed unchanged). Resolution logic itself is unchanged by this plan. |
| T-O2Q-03 | Denial of Service | fail-closed hangup on OTP-only no-code | low | accept | Reuses the existing `_gate_fail_closed` timer — operator-chosen UX, no new surface. |
</threat_model>
