<objective>
Fold an "Inbound DIDs" section into the existing `kv telephony list` command: the actual
VoIP.ms-provisioned numbers the public dials to reach the agent, sourced live from the
VoIP.ms `getDIDsInfo` REST method. It renders FIRST (above the caller-ID mint mappings),
by default, and degrades gracefully when VoIP.ms creds are absent or the API fails —
never aborting the command and never leaking credentials.

Purpose: The shipped `kv telephony list` answers "who gets auto-identified by caller ID"
(DynamoDB mint mappings), the gate secrets, and the gate config — but NOT "what numbers do
people actually call?" Those DIDs are wildcard-routed to the Asterisk edge (`_X.` →
Stasis) and are NOT enumerated in repo config; VoIP.ms is the source of truth.

Output: additive extension to the shipped code (commit 864865f) — a typed VoIP.ms
`getDIDsInfo` helper + a graceful-degradation report field + renderer section + tests. No
new subcommand, no config/router/gate-logic changes beyond relabeling two section headers.
</objective>
