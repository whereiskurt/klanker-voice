Key conventions to mirror exactly (all live in voipms.go):
- Method-name constants: declare each new method as a `voipmsMethod*` const in the
  centralized constants block (getDIDsUSA, getRateCentersUSA, orderDID, cancelDID).
  Do NOT add a second literal base URL — "rest.php" must still appear exactly once
  (TestVoipmsMethodNamesCentralized enforces this).
- Credentials: subcommands call `cfg.resolveVoipmsCreds(c.Context())` then
  `newVoipmsClient(creds)`; never a flag, never printed.
- Error redaction: all HTTP goes through `vc.do()` (or `voipmsDoWithRetry`), whose
  *url.Error unwrap strips the credential-bearing query string from transport errors.
  A non-success envelope already surfaces as `*voipmsStatusError` carrying only the
  safe status/error enums — surface `err` directly; never build error strings from the
  request URL or params.
- Defensive JSON parsing: reuse `voipmsStringField` for map field reads; handle the
  VoIP.ms array-vs-single-bare-object quirk the way ListInboundDIDs does.
- Rendering: plain `text/tabwriter` like telephony.go's Inbound DIDs section
  (NewWriter(out, 0, 2, 2, ' ', 0), tab-separated header + rows, Flush).
</context>
