# Phase 07.1 — Live-Test Polish (Design Spec)

**Date:** 2026-07-07
**Origin:** Kurt's feedback after the first real-phone test of the Phase-7 deploy at voice.klankermaker.ai.
**Scope:** Four quick-win polish fixes + redeploy. Deliberately small; a broader UI revamp is anticipated later, so captions are fixed reasonably, not perfectly.

## 1. TTS pronunciation normalizer

**Problem:** ElevenLabs mispronounces project proper nouns (`meshtk`, `defcon.run`/`DEF CON`, `km`, `CLI`, `Guelph`). A prompt-level hack in `concierge.md` ("avoid glued forms like defcon.run") is unreliable and would pollute captions.

**Approach:** A local `PronunciationTextFilter` implementing pipecat's `BaseTextFilter`, passed via `text_filters=[...]` to `ElevenLabsTTSService`. pipecat applies text filters *after* aggregation and *before* synthesis (`tts_service.py`), so only the spoken text is rewritten. **Captions are unaffected** because the RTVI agent caption is built from `LLMTextFrame` (raw LLM output), upstream of TTS (`rtvi/observer.py::_handle_llm_text_frame`). Keyless, in-process, unit-testable.

**Map (written → spoken):**

| written | spoken |
|---|---|
| `meshtk` | `Mesh Tee Kay` |
| `defcon.run` | `deaf con run` |
| `DEF CON` / `defcon` | `deaf con` |
| `km` | `kay em` |
| `CLI` | `see elle eye` |
| `Guelph` | `Gwelf` |

**Rules:** case-insensitive match; word-boundary anchored so `km` never fires inside `kmv`/`10km`/`km's` (longest-match-first; `defcon.run` before `defcon`). Map lives in config/code, trivially extensible. After landing, **remove** the `concierge.md` glued-form hack so LLM text + captions read naturally.

**Tests:** each mapping fires standalone; no false-positive inside larger tokens; possessives (`km's` → `kay em's`) handled; captions path (LLMTextFrame) proven untouched.

## 2. Double-greeting echo

**Problem:** After the pre-rendered greeting clip, the LLM re-introduces itself on its first turn ("Hey, I'm KPH…") then answers — a double greeting. `greet_first=false` is already correct (server never greets); `concierge.md` already says "don't reintroduce," but the model disobeys.

**Approach:** (a) sharpen the `concierge.md` opening-move instruction, and (b) inject a one-shot first-turn system/developer nudge into the context ("the spoken greeting already played; do NOT greet or introduce yourself — answer the user's first turn directly"). This is the same mechanism class as the old greet-kick, inverted.

**Tests:** first-turn context carries the no-greet nudge; regression that `greet_first` stays false.

## 3. TTS energy retune (tunable)

**Problem:** Welcome reads over-the-top/punchy, then tapers; live TTS uses ElevenLabs defaults (only `speed` was set).

**Approach:** Expose `stability`, `similarity_boost`, `style` (alongside existing `speed`) in `pipeline.toml [tts]`, thread through `config.py` → the ElevenLabs factory (`_build_tts_elevenlabs`) **and** `render_greetings.py`, so the greeting clips and the live voice share identical settings. Re-render greetings (one billed ElevenLabs call). Voice stays `EIDLidDc73oTUWAedVPt` (Burt Fundeck) so the drift guard holds.

**Defaults (from Kurt's lab screenshot):** `speed = 1.0`, `stability = 0.40`, `similarity_boost = 0.85`, `style = 0.10`. All config-tunable; Kurt tunes by ear after deploy.

**Tests:** factory builds voice_settings from the four config fields; render script reads the same fields; drift guard stays green.

## 4. Captions scroll fix (reasonable, not perfect)

**Problem:** Long agent turns clip to the first line via `.caption-text { overflow: hidden }`.

**Approach:** Let the agent caption grow to a max height then scroll within (auto-scroll to newest), preserving the last-exchange-only model. Client-only, no billed render. Kept intentionally simple pending a possible UI revamp.

**Tests:** existing caption reducer tests stay green; a long multi-sentence agent utterance renders fully (grows/scrolls) rather than clipping.
